Lightning Talked About Joybox in RubyMotion Kaigi 2013

May 31st, 2013

Hello! I’m @yonekawa of Joybox Evangelist(Unofficial).

At 5/29, I talked about Game Development of RubyMotion with Joybox in RubyMotion Kaigi 2013.

This is my slide.
https://speakerdeck.com/yonekawa/dive-into-joybox

Sample game for talk.
https://github.com/yonekawa/cannon-gorillas

RubyMotion Kaigi 2013

RubyMotion Kaigi 2013 is first RubyMotion conference in Japan!
Laurent Sansonetti of RubyMotion Developer comes in Japan. He talked about RubyMotion’s roadmap.

It is very exciting conference. The conference has Q&A session to Laurent. @watson1978 talked about RubyMotion in-depth. In Lightning Talk, some one talked about IRC client development by RubyMotion.

My best session is this by @mfks17. https://speakerdeck.com/mfks17/that-objective-c-guy-think-about-rubymotion

Because, I’m also Objective-C guy. RubyMotion teaches Ruby ecosystem for Objective-C guy. I think it is one of the RubyMotion value. We can return Know-how to Objective-C easily because RubyMotion is thin layer.

My motivation

I bought RubyMotion 2 month ago. RubyMotion Kaigi started entries at the same time.
Want to do some interest thing by RubyMotion for Lightning Talk. So, I found blog post about RubyMotion + Cocos2D.
“RubyMotion Game Development” seems very interesting theme.

However, there was many problem. C++ problem, build failure an app crash… I tried many Objective-C wrapper for phyisics engine…

So I found Joybox at that time. It is amazing project. I decided to talking about Joybox.

Joybox

Joybox is awesome but it is not enough now. Especially box2d function is not enough because we need to write Objective-C wrapper.

However, RubyMotion community has a high expectation to Joybox. So, you can do anything for Joybox improvement. You may be able to become a hero by code of several lines. Let’s pull-request! I will support if you contact on twitter :)

I’m dreaming that Rubyist can develop a native game easily with Joybox in a future!
